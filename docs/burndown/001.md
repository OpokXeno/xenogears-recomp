# Burn-down #1 — first run (2026-07-18)

## Symptom

Windowed run (`--game game.toml --bios psxrecomp/bios/SCPH1001.BIN`, launcher
flow, vsync-paced 59.94 fps):

**The game boots.** Human confirmed on screen: launcher → Sony/PS boot logo →
**Xenogears title screen** → New Game → **opening FMV** → intro dialogue text.
Crash when the human pressed "skip text" twice in rapid succession during the
intro dialogue. The runtime's crash_trace captured the fault (guest crash, not
process death); the runtime window stayed up (`exit_origin=sdl_window_close`).

Baseline headless run (`--headless`, implies `--no-launcher`):
`001-first-run.log` — 120 s, no crash, but also no disc/game markers: appears
to idle in the BIOS. The **launcher flow is the working boot path**; the
straight headless boot path is a separate open question (see below).

## Fault data (`001-crash.txt`, `001-report.json`)

```
FAIL-FAST unknown dispatch: addr=0x80019524 phys=0x00019524 ra=0xBFC03D60 a0=0x00000001
```

- Frame ~9644 (~2.7 min in). `exit_reason=atexit`, origin window close.
- `addr=0x80019524` = **game entry PC**; `ra=0xBFC03D60` (BIOS ROM), `a0=1` —
  the BIOS boot-call register signature (as at power-on game handoff).
- `last_func=0x00000F40` (BIOS RAM stub), `last_store_pc=0x8004B568`,
  `epc=0x8004B58C`, `cause=0x400`.
- `epc`/`last_store_pc` both lie inside **`func_8004B54C`** (next fn
  0x8004B694) — a compiled, dispatched game function. Unnamed until Track B.

## Facts established (narrowing the space)

- `func_80019524` **is** compiled (full_19.c) and **is** in the dispatch table
  (5 refs) and **is** seeded — so "unknown dispatch" is a *runtime dispatch-
  state* rejection, NOT a static discovery gap (the crash text's
  "seed it and regen" hint does not apply here).
- Text image guard armed on the local EXE (0x80010000..0x80059800); the fault
  address is inside that range.
- Release build has `PSX_NO_DEBUG_TOOLS`: `recent_fn`, dispatch history,
  xprobe all empty — no execution history available. Diagnostics rebuild
  needed for round 2.

## Hypotheses (ranked)

1. Rapid double-skip raced the guest's event/overlay loader; the guest took an
   error/reset path through the BIOS boot vector back to 0x80019524, and the
   dispatcher fail-fasted on re-entering the entry in a post-boot state.
2. A store in `func_8004B54C` (last_store_pc) dirtied the registered text
   image → guard invalidation → entry re-dispatch rejected.
3. Genuine wild jump to the entry vector from corrupted guest state (several
   GPRs read 0xFFFFFFFF-class values in the dump).

## Next steps (A2)

1. Rebuild with `-DPSX_DEBUG_TOOLS=ON` (or RelWithDebInfo) for
   recent_fn/dispatch history; reproduce the crash deterministically.
2. Check the runtime's native-text predicate for `func_80019524` re-entry
   after boot (`psx_game_text_native_ok` range semantics at pin 678c71f).
3. Inspect `func_8004B54C`'s store target — what guest structure lives at the
   written address; does it overlap the registered text image?
4. Secondary: why does the straight `--headless` boot idle in the BIOS while
   the launcher flow boots the game? (Different disc-seed path?)
5. Track B symbols for `func_8004B54C` once B1 lands.

## Reproduction command

```sh
./build/XenogearsRecomp --game game.toml --bios psxrecomp/bios/SCPH1001.BIN
# launcher → launch game → New Game → watch FMV → skip intro text twice fast
```

## Addendum — freeze dump evidence (`001-freeze.json`)

A freeze-heartbeat dump also fired at **frame 1194** (~20 s, BIOS→game
handoff window): `current_func=0x1FC03CF0`, `last_store_pc=0xBFC03D04`,
`exception_entries=1285` (CD IRQ burst — EXE loading), `dispatch_count=1`.
The loop site sits in the same BIOS boot-caller region
(0xBFC03Cxx–0xBFC03Dxx) that later appears as the crash's `ra=0xBFC03D60`.
I.e. the BIOS EXE-loader/boot-caller is implicated at both ends: it was
heartbeat-flagged during the handoff, and the fatal re-entry at frame 9644
came *from* it. Strengthens hypothesis 1 (boot-vector re-entry rejected).
